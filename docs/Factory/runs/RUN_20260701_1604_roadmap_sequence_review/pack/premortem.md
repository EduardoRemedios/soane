## Version

v1

## Change Log

- v1 (2026-07-01): Initial Stage E premortem.

## Failure Scenarios

1. The roadmap remains a fork instead of a sequence.
2. Workspace Shell starts before the coding workflow is navigable.
3. Live adapters start before deterministic workflow boundaries are inspectable.
4. Persistence is chosen before memory-provider evaluation and workflow usage evidence.
5. Brownfield multi-repo complexity is deferred too long and surprises product design.

## Mitigations

- Add explicit near-term sequence.
- Put workflow wrapper before Workspace Shell.
- Put live adapter evaluation after workflow wrapper.
- Put memory-provider evaluation before persistence architecture.
- Add Brownfield multi-repo proof before product surface.
