## Purpose

Strengthen Soane's constitutional and canonical architecture so its language remains accurate as Project Memory ingestion, delegated agents, cross-project retrieval, and decision support become real. [SOURCE:RAW]

## Goal

Define a coherent epistemic and governance model for claims, project understanding, delegated judgement, decision outcomes, memory rights, knowledge scope, Markdown authority, and product success without prematurely changing runtime schemas. [SOURCE:RAW]

## Requirements

- Project Memory must be the governed system of record for the Workspace's current understanding, not the ultimate authority for external reality or source-system records. [SOURCE:RAW]
- A Claim or Knowledge Statement must distinguish an assertion from Evidence, Verification, Decision, and accepted fact status. [SOURCE:RAW]
- Facts must be treated as reviewed epistemic status supported by provenance and appropriate evidence, not as an unqualified object label. [SOURCE:RAW]
- Humans must retain accountability and governed authority while agents may exercise explicit, bounded, revocable judgement under authority, policy, evidence, budget, and escalation conditions. [SOURCE:RAW]
- Decision records must support outcome review linking expected consequences, observations, assumption validity, learning, and amendment or reversal. [SOURCE:RAW]
- Memory governance must cover capture purpose, consent where required, scope, access, correction, export, retention, deletion, redaction, and revocation. [SOURCE:RAW]
- Cross-project knowledge must fail closed and require explicit governed promotion with provenance and conflict preservation. [SOURCE:RAW]
- Decision support must frame objectives, alternatives including status quo, criteria, causal assumptions, predictions, trade-offs, dissent, confidence, and reversibility proportionally to consequence. [SOURCE:RAW]
- Markdown must distinguish authored authority, generated projection, and curated round trip; generated output must not overwrite authored meaning or silently promote memory. [SOURCE:RAW]
- Product success measures must evaluate trustworthy context, uncertainty reduction, decision learning, constraint adherence, provenance, and human attention rather than artifact volume. [SOURCE:RAW]
- Enterprise scaling and voice language must preserve accessibility, privacy, consent, identity, tenancy, and policy realities. [SOURCE:RAW]
- Current implementation status and all deferred runtime changes must remain explicit. [SOURCE:REF:docs/PROJECT_STATE.md]

## Non-Goals

- runtime object-registry or fixture changes
- persistence, migration, API, or UI design
- automatic ingestion, extraction, promotion, or cross-project sharing
- replacement of Aegis authority/proof, Factory V3 mission governance, Temper runtime, Sentinel discovery, or Harmony regulated runtime
- broad stylistic rewrite of canonical documents

## Principles

- Preserve source-system authority and portfolio ownership.
- Make epistemic status explicit and revisable.
- Capability does not imply authority; delegation does not erase accountability.
- Privacy and access constraints propagate with knowledge.
- Contradictions remain visible until governed resolution.
- Constitutional doctrine should be durable; implementation detail should remain deferred until evidence requires it.

## Roles

- Workspace: owns human-facing project understanding, context assembly, and memory views.
- Project Memory: records governed understanding and external references without becoming external authority.
- Human or responsible authority: remains accountable for governed decisions and delegations.
- Agent: may reason and act only within explicit delegated bounds.
- Source systems: remain authoritative for records they own.

## Acceptance Criteria

- The scoped documents use compatible definitions of Project Memory, Claim, fact status, delegation, Decision Review, Knowledge Scope, and Markdown modes.
- The amendments preserve existing product boundaries and do not imply runtime implementation that does not exist.
- Cross-project sharing defaults to denied until a governed promotion path is satisfied.
- Memory rights and access propagation are constitutional concerns rather than incidental implementation notes.
- Success measures are observable and resistant to artifact-volume gaming.
- Knowledge lint, context-index refresh, pack lint, and the repository test suite pass.

## Open Questions

- NON-BLOCKING: Whether the future runtime type is named `Claim` or `KnowledgeStatement`; doctrine may define Claim and defer registry naming.
- NON-BLOCKING: Exact organisational identity and policy contracts remain future integration work.

## Go Or No-Go Rule

Go only if amendments improve semantic precision without transferring portfolio ownership, asserting unimplemented runtime capability, weakening human accountability, or authorizing automatic knowledge promotion.
